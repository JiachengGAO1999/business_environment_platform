# 视觉风格基准

本项目后续页面、组件和交互开发，默认以 `docs/preview.png` 与当前实现为视觉基准。目标不是做装饰型大屏，而是形成适合营商环境情报分析的克制、可信、可长时间阅读的工作界面。

## 关键词

- 温润：米白纸面、低饱和绿色、暖灰文字，避免冷硬后台感。
- 克制：少用强色块和重阴影，重点依靠留白、层级、细线和内容密度建立。
- 可信：报告、证据、筛选、审核等页面应像研究工作台，而不是营销页。
- 清晰：每页必须有一个主重点，辅助信息围绕主任务组织。

## 色彩

基础色使用 `src/index.css` 中的 token，不在页面中随意新增高饱和主题色。

- 背景：`background` / `parchment`，米白到浅暖灰的轻微层次。
- 卡片：`card` / `ivory`，接近纸面，透明度可轻微叠加。
- 主色：`brand-*`，鼠尾草绿，用于导航激活、关键图标、地图点、主按钮。
- 提醒色：`accent-amber-*`，用于中高风险、重点提示、待复核。
- 危险色：`danger-*`，只用于严重风险、错误、破坏性反馈。
- 文本：`foreground` 用于主文本，`muted-foreground` 用于说明和元信息。

避免一页被单一强色占满；避免大面积深蓝、纯黑、紫色渐变、强科技蓝。

## 字体与排版

- 字体沿用 `Geist Variable`，不额外引入装饰字体。
- 页面标题使用 `font-medium`，不要过粗。
- 正文与说明保持 `leading-6` 到 `leading-7`，让中文阅读更舒展。
- 字号层级应克制：页面主标题可用 `text-3xl`，普通页标题用 `text-2xl`，卡片内标题多用 `text-sm` 或 `text-base`。
- 不使用负字距；全局 `letter-spacing` 保持 0。

## 组件风格

- 卡片统一使用 8px 左右圆角、细边框、低阴影。
- 不嵌套卡片；复杂信息用分区、细线、列表、表格组织。
- Badge 使用低饱和浅底色，避免鲜艳红绿黄。
- 图标优先使用 `lucide-react`，尺寸通常为 14px 到 18px。
- 主按钮使用品牌绿，次级按钮使用 outline 或 ghost。
- 页面级重点区域使用 `.page-panel`。
- 小节眉标使用 `.section-eyebrow`。

## 页面组织

- 每页先定义一个核心任务：观察态势、筛选资料、阅读报告、审核结论等。
- 工作台页面应先给出主叙事或关键状态，再给 KPI 与列表。
- 表格页保持密度，但筛选栏和表格边界要轻，不要做厚重工具栏。
- 报告页优先保证长文阅读和证据追踪，不做过多装饰。
- 全球大屏页应表达全球态势和监测焦点，但仍保持柔和、清晰、可读。

## 地图与大屏

全球大屏基准是“情报态势室”，不是高对比炫技屏。

- 地图用浅色线框或低透明度地理形状。
- 监测点用品牌绿或琥珀色强调，不使用闪烁动画。
- 右侧焦点面板展示当前国家的风险、机会、摘要和报告入口。
- 数据指标用小型 signal card，避免铺满屏幕的大数字墙。

## 开发约束

- 新页面优先复用 `PageShell`、`Card`、`RiskBadge`、`DimensionTag`、`StatusBadge` 等共享组件。
- 需要新增颜色时，先判断能否放入 `src/index.css` token；不要在多个页面散落相近颜色。
- Tailwind class 可以局部使用任意值，但同一语义重复出现时应沉淀为 token 或共享组件。
- 新增页面完成后至少运行 `pnpm build`；涉及组件样式大改时同时运行 `pnpm lint`。
