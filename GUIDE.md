# 开发 GUIDE

本项目后续开发以“真实来源可追溯、六维模型可解释、专家审核可闭环”为主线。视觉基准见 `docs/design-style.md`，产品与数据流基准见本文档。

## 当前目标

在保留巴西、阿联酋 mock 演示数据的基础上，优先把“沙特阿拉伯”替换为真实数据流样板：

```
来源注册表 → 每日抓取 → 正文清洗 → 六维筛选 → 证据抽取 → 报告生成 → 专家审核
```

沙特真实数据流 v0.1 先采用项目内 TypeScript 数据包承载真实来源样本，后续可替换为后端 crawler + database。

## 六维营商环境模型

六维模型来自 `docs/营商环境构成.docx`：

1. 政治举措及地缘环境
2. 金融投资环境
3. 产业政策及市场环境
4. 合规与法治环境
5. 新质生产力与科创环境
6. 社会人文环境

所有真实数据与 mock 数据均统一使用上述六维标签。旧七维模型已经退出运行时数据、筛选和报告结构。

## 来源注册表

真实来源不由模型自由搜索决定，而由人工筛选后的注册表驱动。沙特来源注册表来自 `docs/沙特营商环境网站.xlsx`，代码入口为：

- `src/data/sourceRegistry/saudi.ts`

每个来源至少包含：

- 国家
- 六维模型维度
- 网站名称
- URL
- 优先级标签：`紧急` 或 `重要`
- 来源类型
- 建议抓取策略

## 真实数据包

沙特 v0.1 的真实数据包位于：

- `src/data/real/saudi.ts`

数据包包含：

- `saudiRealSources`：真实来源资料，写入 `SourceDocument`
- `saudiRealEvidence`：可点击证据片段，写入 `EvidenceSnippet`
- `saudiRealReport`：基于六维模型生成的沙特报告

当前 `src/data/mock/index.ts` 会把原 mock 沙特数据替换为该真实数据包，其他国家仍走 mock。

## 后续接入 crawler 的替换点

后端 crawler 成熟后，应保持前端数据结构不变，只替换数据来源：

1. 从 `sourceRegistry` 读取每日抓取任务。
2. 抓取网页、PDF、公告列表或开放数据页面。
3. 对正文做去重、语言识别、发布时间解析和正文清洗。
4. 生成 `SourceDocument`。
5. 调用 Agent 服务完成相关性评分、维度归类和证据抽取。
6. 生成或更新 `AnalysisReport`。

## 验收标准

沙特真实数据流的最低可验收效果：

- 资料库中沙特资料标记为 `real`。
- 智能筛选中沙特资料使用六维模型标签。
- 报告详情中沙特报告以六维章节组织。
- 报告正文中的证据编号可以点击并展示真实来源 URL。
- 审核中心可以对沙特报告继续执行段落级审核。

## 开发检查

完成数据或页面变更后运行：

```bash
pnpm build
pnpm lint
```

`shadcn/ui` 组件中可能存在 Fast Refresh warning；只要没有新增 error，可以继续提交。
